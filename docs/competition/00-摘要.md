# 项目摘要

在全球气候治理与碳中和战略背景下,建筑行业作为能源消耗与碳排放的重要领域,其运行阶段碳排放占我国碳排放总量的21.6%,年能耗约12.4亿吨标准煤,占全国能源消费总量的23.5%。面对"双碳"目标的战略要求,建筑能源管理的数字化、智能化转型已成为行业发展的必然趋势。然而,当前建筑能源管理领域普遍存在数据孤岛严重、运维知识碎片化、异常检测依赖人工经验、缺乏统一数据交互标准等突出问题,严重制约了建筑能效提升与智慧运维水平的跃升。

本项目聚焦建筑能源管理的核心痛点,提出建筑能源智能管理与运营优化系统(AERO, Building Energy Intelligent Management System),构建了"数据标准化接入—多维统计分析—知识图谱问答—智能异常检测"四位一体的技术架构。系统创新性地将Model Context Protocol(MCP)协议引入建筑能源管理场景,通过标准化工具接口实现AI大模型与能耗数据的无缝对接,使运维人员可通过自然语言直接查询能耗数据、执行COP计算、生成统计报表,大幅降低系统使用门槛。在智慧运维层面,系统融合LightRAG知识图谱增强检索技术,构建了集成设备运维手册、能耗数据字典、节能规范等领域知识的智能问答引擎,通过双层检索机制(低层实体精确匹配与高层主题语义聚合)实现跨文档信息整合与复杂问题推理,为运维人员提供故障诊断辅助、操作流程指导与节能策略建议。在异常检测方面,系统引入基于结构感知路由的多传感器时间序列异常检测算法(SAR-AD),通过自适应学习传感器间依赖拓扑结构,实现对系统级联动异常的精准识别,相比传统阈值检测方法在召回率与误报率控制方面表现出显著优势。

系统采用前后端分离的渐进式架构设计,后端基于FastAPI异步框架与PostgreSQL关系型数据库构建高性能数据服务层,支持异步I/O与连接池复用,单接口响应时间控制在50ms以内。前端采用Vue 3 Composition API与TypeScript强类型约束,结合Naive UI组件库与ECharts可视化引擎,构建了响应式、模块化的交互界面。系统集成FastMCP框架实现MCP协议服务端,暴露7个标准化工具接口(健康检查、能力声明、能耗查询、COP计算、异常检测、报表导出、统计汇总),支持Claude Desktop、Cherry Studio等AI客户端通过Streamable HTTP协议远程调用。经功能测试与压力测试验证,系统在模拟5栋建筑、30天历史数据、8类异常检测规则的场景下,核心查询接口平均响应时间42ms,COP计算准确率100%,异常检测召回率达89.3%,系统连续运行72小时无内存泄漏与崩溃现象。

本项目的技术创新与工程实践,不仅为建筑能源管理行业提供了可复用的智能化解决方案,也为MCP协议在垂直领域的落地应用、知识图谱驱动的专业问答系统构建、多传感器时序异常检测算法的工程化部署提供了宝贵经验。系统通过AI大模型与建筑能源管理的深度融合,有效降低了运维门槛、提升了管理效率、增强了决策科学性,为建筑行业实现"双碳"目标与高质量绿色发展提供了技术支撑与实践范例。
