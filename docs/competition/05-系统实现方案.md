# 5 系统实现方案

本章详细介绍AERO系统的工程实现方案，包括后端架构、前端界面设计、数据库实现和API接口设计。

## 5.1 软件系统整体框架

系统采用前后端分离的B/S架构，后端基于FastAPI提供RESTful API和MCP服务，前端基于Vue 3构建单页应用（SPA）。

## 5.3 数据库实现

### 5.3.1 数据库选型

选择PostgreSQL作为核心数据存储引擎，主要考虑以下因素：

| 特性 | PostgreSQL | MySQL | SQLite |
|------|-----------|-------|--------|
| 时序数据支持 | 优秀（支持时间戳索引、窗口函数） | 良好 | 一般 |
| JSON支持 | 原生jsonb类型 | JSON类型 | 无 |
| 并发能力 | MVCC，支持高并发读写 | 行级锁 | 文件级锁 |
| 扩展性 | 丰富的扩展生态（如TimescaleDB） | 有限 | 有限 |
| 异步驱动 | asyncpg（高性能） | aiomysql | aiosqlite |

### 5.3.2 ORM模型设计

采用SQLAlchemy 2.0异步ORM，基础模型设计：

```python
# base.py - 基础模型
class TimestampMixin:
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    updated_at: Mapped[datetime] = mapped_column(
        default=func.now(), onupdate=func.now()
    )

class Base(DeclarativeBase, TimestampMixin):
    pass
```

关键模型：

**Building模型**：建筑实体，使用UUID主键，building_id唯一约束
**EnergyRecord模型**：能耗记录，核心时序数据表，(building_id, timestamp)联合索引
**Equipment模型**：设备实体，关联建筑
**AnomalyEvent模型**：异常事件，多维度索引

### 5.3.3 数据库迁移管理

使用Alembic进行版本化数据库迁移管理，确保数据库Schema变更的可追溯性和可回滚性。

```bash
# 生成迁移脚本
alembic revision --autogenerate -m "描述"
# 执行迁移
alembic upgrade head
# 回滚
alembic downgrade -1
```

## 5.5 关键技术实现细节

### 5.5.1 COP计算引擎

```python
# utils/cop.py
def calculate_cop(
    hvac_supply_temp: float,    # 供水温度 ℃
    hvac_return_temp: float,    # 回水温度 ℃
    hvac_flow_rate: float,      # 水流量 m³/h
    hvac_kwh: float,            # HVAC能耗 kWh
    period_hours: float         # 时段小时数
) -> float:
    # 制冷量 = 流量 × 比热容系数 × 温差
    q_cooling = hvac_flow_rate * 1.163 * abs(hvac_return_temp - hvac_supply_temp)
    # 输入功率
    w_input = hvac_kwh / period_hours
    # COP = 制冷量 / 输入功率
    return q_cooling / w_input if w_input > 0 else 0
```

### 5.5.2 阈值检测引擎

```python
# detection/threshold.py
class ThresholdDetector(AbstractDetector):
    def detect(self, records) -> List[AnomalyEvent]:
        anomalies = []
        # 1. 能耗突增检测
        anomalies += self._detect_energy_spike(records)
        # 2. COP异常检测
        anomalies += self._detect_low_cop(records)
        # 3. 温度偏差检测
        anomalies += self._detect_temp_deviation(records)
        return anomalies
```
