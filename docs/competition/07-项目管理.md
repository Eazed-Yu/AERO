# 7 项目管理

## 7.1 团队组织与分工

项目团队采用扁平化组织结构,成员间职责明确、协作紧密。项目负责人统筹项目整体进度,负责系统架构设计、技术方案制定、文档编写与对外汇报,协调团队成员间的工作分配与任务衔接。后端开发负责FastAPI后端服务开发,包括数据库模型设计、业务逻辑实现、RESTful API开发、MCP协议集成、异步性能优化等工作,确保后端服务的高性能与稳定性。前端开发负责Vue 3前端应用开发,包括页面组件实现、ECharts可视化图表集成、Pinia状态管理、API接口调用、用户交互优化等工作,确保前端界面的美观性与易用性。算法研究负责时间序列异常检测算法研究与实现,包括SAR-AD模型训练、LightRAG知识库构建、检测策略优化等工作,确保智能算法的准确性与实用性。指导教师提供技术指导与项目审核,定期参与项目评审会议,对技术方案与实现质量提出指导意见。

团队采用敏捷开发方法,以两周为一个迭代周期,每个迭代包括需求分析、设计开发、测试验证、评审总结四个阶段。团队每日进行站会,同步工作进展与遇到的问题,每周进行技术评审会议,讨论技术难点与解决方案。团队使用Git进行版本管理,采用Feature Branch工作流,main分支为稳定发布版本,feature/*分支为功能开发分支,fix/*分支为Bug修复分支,每个功能开发完成后通过Pull Request合并至main分支,合并前需经过代码审查与测试验证。

## 7.2 开发流程与时间管理

项目开发周期为12周,分为五个阶段推进。第一阶段为需求分析与系统设计(第1-2周),主要工作包括赛题需求深入分析、技术方案调研与选型、系统架构设计、数据库模型设计、原型设计,交付物为需求分析文档与系统设计文档。第二阶段为数据层建设(第3-4周),主要工作包括能耗数据集设计与构建、数据库Schema实现、数据导入导出功能开发、数据清洗与标准化,交付物为数据集、数据库Schema、导入功能。第三阶段为核心功能开发(第5-8周),主要工作包括查询统计模块开发(多条件查询、聚合统计、COP计算)、异常检测模块开发(阈值检测器、ML检测器接口)、LightRAG知识引擎集成、智能问答功能实现,交付物为查询接口、统计功能、可视化图表、知识库、问答功能、异常检测。第四阶段为MCP集成与系统整合(第9-10周),主要工作包括MCP工具设计与开发、前后端联调、系统集成测试,交付物为MCP集成、前后端联调。第五阶段为优化与文档(第11-12周),主要工作包括性能优化与Bug修复、项目文档编写、演示视频制作、最终测试与验收,交付物为完整文档、演示视频、数据集。

项目时间管理采用里程碑驱动模式,每个阶段结束时设置里程碑检查点,评估阶段目标完成情况与交付物质量,及时调整后续计划。项目使用甘特图进行进度可视化管理,通过任务依赖关系分析关键路径,识别进度风险,采取措施保障项目按期交付。

## 7.3 质量保障措施

项目建立了多层次的质量保障体系。代码规范层面,后端代码遵循PEP 8规范,使用Black进行代码格式化,使用Pylint进行静态代码检查,前端代码遵循ESLint规范,使用Prettier进行代码格式化,确保代码风格一致性与可读性。类型检查层面,后端使用Pydantic进行运行时类型验证,前端使用TypeScript进行编译时类型检查,减少类型错误导致的运行时异常。接口文档层面,FastAPI自动生成OpenAPI文档,提供交互式API测试界面,前后端开发人员通过文档进行接口对接,减少沟通成本。数据库迁移层面,使用Alembic进行版本化管理,每次Schema变更生成迁移脚本并纳入版本控制,确保数据库变更的可追溯性与可回滚性。

测试层面,项目建立了单元测试、集成测试、系统测试三级测试体系。单元测试覆盖核心业务逻辑函数,如COP计算函数、阈值检测函数等,使用pytest框架编写测试用例,确保函数级别的正确性。集成测试覆盖API接口,使用FastAPI的TestClient进行接口测试,验证请求响应的正确性与异常处理的健壮性。系统测试覆盖端到端业务流程,通过手工测试与自动化测试相结合的方式,验证系统功能的完整性与用户体验的流畅性。

## 7.4 风险管理与应对策略

项目识别了技术风险、进度风险、资源风险三类主要风险。技术风险包括MCP协议集成复杂度高、LightRAG知识图谱构建效率低、SAR-AD模型训练数据不足等,应对策略为提前进行技术预研与原型验证,遇到技术难点及时寻求指导教师与社区支持,必要时调整技术方案降低复杂度。进度风险包括功能开发时间超预期、测试发现重大缺陷需返工等,应对策略为采用敏捷开发方法,小步快跑快速迭代,每个迭代结束时进行评审,及时发现问题并调整计划,预留缓冲时间应对突发情况。资源风险包括团队成员时间冲突、硬件资源不足等,应对策略为合理分配任务,避免关键路径上的单点依赖,必要时调整任务优先级,确保核心功能优先完成。

项目建立了周报制度,每周汇总项目进展、遇到的问题、下周计划,通过周报及时发现风险并采取应对措施。项目建立了问题跟踪机制,使用GitHub Issues记录Bug与待办事项,明确责任人与截止时间,确保问题得到及时解决。项目建立了文档管理机制,所有设计文档、技术文档、测试文档纳入版本控制,确保文档的完整性与可追溯性。
